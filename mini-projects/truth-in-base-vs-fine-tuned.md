# Geometry of Truth in Base vs Fine-Tuned Models

<p style="opacity: 0.6; font-size: 0.9em; margin-bottom: 2rem;">Dec. 2025</p>

## Summary
### Goals
This mini project aims to investigate whether linear representations of truth emerge or become more pronounced as a result of instruction fine-tuning. Motivated by [*The Geometry of Truth*](https://arxiv.org/pdf/2310.06824), which demonstrates that ”linear representations of truth emerge with scale”, I initially thought that fine-tuning for aligned responses might similarly emphasize a truth-related direction in activation space. Specifically, I hypothesized that such a direction would be easier to identify with linear probes and more effective for causal steering in fine-tuned models.

The task focused on here is classifying basic true and false statements across multiple domains and worked with Llama-3.1-8B and Llama-3.1-8B-Instruct, which undergoes an instruction fine-tuning on top of the base model.

### High-Level Takeaways
*   **Truth is linearly decodable in both models, but more robustly in Instruct.**  
    Linear probes achieve high accuracy for both base and instruct activations, but probe performance is more stable across layers/datasets in the instruct model, whereas the base model peaks in the middle layers and then degrades in performance.
*   **“Truth directions” are more consistent across datasets in Instruct fine-tuned.**  
    Cosine similarity between probe directions trained on different domains is higher in the instruct fine-tuned model, suggesting a more uniform (less dataset-specific) linear feature.
*   **Causal steering is asymmetric, and mostly shows up with Instruct fine-tuned derived directions.**  
    Base-derived probe directions produced no significant steering. Instruct-derived directions yield reasonable effects, mainly steering true → false over false → true.
*   **Cross-model transfer suggests fine-tuning reweights an existing feature rather than creating a new one.**  
    Fine tune-derived directions can steer the base model, but base-derived directions don’t steer the instruct model, consistent with fine-tuning making an existing truth-relevant direction easier to isolate.
*   **Control experiments with random directions show steering effects are direction-specific.**  
    To verify that steering effects were not due to chance, I repeated the causal analysis using randomly sampled directions matched in scale and intervention site.

Overall, I found that instruction tuning increases linear separation and consistency of a truth direction, but steering effects are weak and asymmetric, so the evidence does not support the hypothesis that “truth newly emerges.” A major limitation to this study is that the instruct model can have higher baseline task performance (at this model size), which can inflate apparent linear separability and steering (although I tried to mitigate the impact of this by filtering to statements that either both models could correctly answer or both incorrectly answer).

Similar to the original paper, we see importance given to the final token (which makes sense as this is what directly feeds into the final logit calculation), and to key tokens that form the meaning of the sentence (e.g. the numbers in the larger than / smaller than dataset or the country/city in the cities dataset). We also see some evidence for the group of activations referred to as summarization behavior. This is the information stored at the end of sentence token around layers 11-14 in both models and this is where we focus our probes and intervention on.

## Experiment 1 - Probing
To test for linear representations, I try to separate true and false statements using linear probes on the end of statement token activations. I used logistic probes (LR) and mass mean probes (MM).

We can see an identifiable linear boundary emerging around layer 12. For the base model that boundary fades as we get to later layers, but not for the instruct fine-tuned model. This is consistent with the PCA plots. My hypothesis is that instruction fine-tuning reinforces the downstream importance of this “truth” subspace, making it less susceptible to transformation in later layers. In contrast, the base model appears to transform representations in later layers in a way that reduces linear separability of the truth, but might still be separable in a nonlinear way.

### Observations
We can see that the instruct model has higher accuracy than the base model on all (train, test) pairs except for 2 on the Mass Mean Probe and the oracle probe achieves perfect accuracy. In particular, I want to highlight a few points
*   **Interpretability Flaw:** Interpreting probe and steering results is complicated by the fine-tuned model having a higher raw accuracy than the base model on some datasets. Take the extreme, if a model can't identify true from false statements on a dataset, we can’t expect simple linear probes on its activations to reliably recover a truth signal. neg_sp_en_trans appeared to be a clear outlier in the baseline gap, so I excluded it from Experiment 2 (including it did not qualitatively change the results). Furthermore to equalise baselines, I filtered the remaining 5 datasets for statements that either both models could correctly identify or both incorrectly identify - net removing 10.2% of the statements. (This does not fully resolve the issues because the fine tuned model might be more confident on its classification than the base model, but this problem is hard to solve completely because fine tuned models in general will have better baselines)
*   **Generalization:** Probes trained on only unnegated statements (like `cities`) generalize better for the instruct model. I think this is because when the dataset is simple, there are likely correlated features such as "often appears together" that act as a proxy to truth, which might be more strongly represented in the base model but that proxy does not generalise well. 
*   **Probe Types:** Differences between models are most clearly seen on the LR Probe, which could be because truth proxy directions are not orthogonal to the truth direction, which affects the LR optimal direction more than the MM Probe (see discussion in section 5.1 of the paper) and the instruct model activations surface a general truth direction better.

## Experiment 2 - Causal Analysis
Succinctly, I used normalised probe directions derived in the experiments above to steer the models to respond with True to False statements and vice versa. Normalised so that:

$$\text{probe_prob_prediction}(\text{average false activations} + \text{probe direction}) = \text{probe_prob_prediction}(\text{average true activations})$$

The intervention site was the end of statement token and hyper parameter search was conducted on intervention strength and layers (settling on layers 10-14 with intervention strength 2). To measure the effectiveness of the steering, I used the NIE metric defined in the paper.

</figure>

### Observations
*   **Asymmetric Control:** No model exhibits bidirectional control, but the fine-tuned model is more responsive to the added vector than the base model.
*   **Probe Effectiveness:** As found in the paper, the MM Probe is more effective than the LR Probe. The MM Probe direction is essentially the same as the typical method of using the difference of activation means as the steering vector. However, we were not able to replicate the strength of their findings with these models.
*   **Directionality:** The fine tuned model shows significant NIE when trying to steer true statements to False, but not in the reverse direction. This pattern on true-> false being stronger than false -> true also shows in the paper’s NIE results.
*   The observed NIE from the steering on the fine tuned model is highly unlikely under a null distribution induced by random directions at the same intervention site and scale (tested with n=200 random vector directions. For understanding purposes, I included an example in the table)

### Cross-Model Steering
To test if the base model was actually steerable at this intervention site, I tried to steer each models with the probe direction from the *other* model.

<div class="research-grid full-width">
  <figure>
    <img src="/assets/figures/geometry_of_truth/8B_cross_model_steering.png" alt="Cross-Model Steering Results">
    <figcaption>Cross-Model Transfer: Instruct-derived directions steering the Base model.</figcaption>
  </figure>
</div>

**Observations:**  
The base model is receptive to steering! The vector can steer the base model to classify true statements as false, just as in the fine tuned model. Interestingly, unlike the fine tuned model, the direction shows some signs of steering the base model to classify a false statement as true.

The fact that a steering direction extracted from the instruction-tuned model transfers to the base model suggests that instruction tuning does not introduce a new truth-related circuit and linear representation does not “emerge”. Furthermore, I found that the base and fine tuned probe directions on average have a cosine similarity of 48%.

Instead, it is more likely that the feature is already present in the base model but might be more distributed or entangled, such that simple linear probes trained on model activations couldn’t isolate it. The fine tuning appears to reorganize or concentrate this feature at the end of sentence token, making it more linearly identifiable (for the probe).

A potential explanation of why true to false is easier than false to true is that the model represents truth as a high-confidence state that requires consistent support, while falsity comes from a collapse in confidence, which can be induced by perturbing fewer control signals.

## Conclusions
In this work, I investigated how instruction fine-tuning affects the linear identifiability and steerability of truth-related features in LLMs.
*   **Experiment 1** showed that linear probes trained on instruction-tuned model activations generalize more reliably. **Cosine similarity analysis** further showed that probe directions are more aligned in the instruction-tuned model, suggesting increased consistency.
*   **Experiment 2** causal interventions revealed bidirectional control was not observed. While the instruction-tuned model exhibited larger effects, cross-model steering experiments demonstrated that directions extracted from the instruction-tuned model can also causally influence the base model. This indicates that instruction fine tuning does not introduce a new truth circuit.

These results suggest that instruction tuning primarily affects *how* truth-related information is used, rather than *whether* it is represented. Overall, this work supports a view in which instruction tuning reweights existing representations, increasing their linear separability and causal effects, but does not cause a new truth representation to “emerge”, proving my initial hypothesis incorrect.

---

## Limitations and further work
*   **Baseline performance:** Differences in model accuracies complicate interpretations. Matches baselines in larger models might help.
*   **Scope:** Only instruction/alignment fine-tuned LLaMA models were studied.
*   **Compute:** Limited model sizes and hyperparameter sweeps.
*   **Task Simplicity:** Focused on binary true/false; more complex reasoning tasks may differ.
*   **Prompt Formatting:** Used “{statement} This statement is:”. Chat-style prompts might yield different results.
*   **Locality:** Truth-related information may be distributed across more tokens than just the end-of-sentence.
